# CLAUDE.md — Project Instructions for Claude

## What this project is
RMIT UP Financial Accounting Dashboard — a Streamlit + Python + SQLite app built as a portfolio piece for a **Financial Accountant job interview**. The goal is to demonstrate accounting knowledge, Python fluency, and dashboard design in a single deployable app.

## Tech stack
- **Frontend:** Streamlit (deployed on Streamlit Cloud)
- **Backend:** SQLite (auto-generated by `generate_data.py` on startup)
- **Data:** Synthetic FY2026 dataset — RMIT UP chart of accounts, GL, AR, AP, payroll, FBT, GST
- **Repo:** GitHub → Streamlit Cloud (push to `main` → auto-redeploy, or manual reboot)

## App structure
- All UI lives in `app.py` (single file, ~3800+ lines)
- All synthetic data generation lives in `generate_data.py`
- `data/` folder holds CSV source files and the SQLite DB (`finance_data.db`)
- Navigation is a sidebar radio with 11 pages

## Key architectural decisions (don't change without discussing)
- **Global sidebar** holds: Period controls (View Type / FY / Month / Quarter), Region & Cost Centre filters, Tax Rate Configuration (locked by default, ✏️ to unlock)
- **Period variables** (`period_start`, `period_end`, `selected_period`, `period_label`, `REPORT_DATE_DYN`, `prev_period`, `_kpi_lbl`, `_pl_period_note`) are all computed **once in the sidebar** and used by every page — do not recompute inline per page
- **Accounting Equation Check** sources from Balance Sheet totals (not GL directly) so it always = $0 by construction
- **FBT Type 1 gross-up** hardcoded at 2.0802 (ATO FY2026); Type 2 = 1/(1-FBT rate)
- **DPO** = weighted average (days × amount) / total, paid invoices only
- CSS in global `<style>` block at top of `app.py` — `[data-testid="stSidebar"] div` must NOT have `color: #FFFFFF !important` as it breaks selectbox text visibility

## What the user cares about
- App must look professional enough to demo in an interview
- Accounting logic must be technically correct (AASB standards, ATO rates)
- Changes should be minimal and targeted — no refactoring, no extra features unless asked
- Push + Streamlit Cloud reboot is required to deploy any change

## What to avoid
- Don't add features, pages, or sections unless explicitly asked
- Don't use `display:flex` in `st.markdown` HTML — Streamlit doesn't render it reliably
- Don't apply `color: #FFFFFF !important` to `div` in sidebar CSS — breaks widget text
- Don't create new files (docs, readmes, etc.) unless asked
- Don't amend commits — always create new ones
- Don't touch `generate_data.py` unless the user specifically asks about data

## Deployment workflow
1. Edit `app.py` (and/or `generate_data.py` / `data/*.csv` if needed)
2. `git add <files>` → `git commit -m "..."` → `git push`
3. Streamlit Cloud auto-redeploys, or user clicks **Reboot app** in the three-dot menu
4. Shell is not always available in this environment — provide git commands for user to run manually when needed
